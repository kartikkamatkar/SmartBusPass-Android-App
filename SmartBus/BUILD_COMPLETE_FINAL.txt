THE MAIN ISSUE: Invalid Attribute Names
────────────────────────────────────────

RelativeLayout DOES NOT have these attributes:
  ❌ android:layout_alignBottomOf
  ❌ android:layout_alignEndOf
  ❌ android:layout_alignTopOf

These attribute names NEVER existed in the Android framework. The confusion likely
came from:
  • ConstraintLayout which HAS similar attributes
  • Misunderstanding of RelativeLayout syntax

CORRECT RelativeLayout Attributes:
  ✅ android:layout_toLeftOf="@+id/view"
  ✅ android:layout_toRightOf="@+id/view"
  ✅ android:layout_toStartOf="@+id/view"
  ✅ android:layout_toEndOf="@+id/view"
  ✅ android:layout_alignParentTop="true"
  ✅ android:layout_alignParentEnd="true"

OUR SOLUTION: Use Better Layouts
──────────────────────────────────

For Simple Overlays (badges, buttons):
  → Use FrameLayout with android:layout_gravity

For Complex Layouts:
  → Use ConstraintLayout with app:layout_constraint*

Why?
  • FrameLayout is simpler and cleaner
  • ConstraintLayout is more flexible
  • No confusing attribute naming

EXAMPLE 1: Camera Badge (FrameLayout)
──────────────────────────────────────

BEFORE (❌ Invalid):
  <RelativeLayout>
    <ImageView android:id="@+id/iv_pic" ... />
    <View
      android:layout_alignBottomOf="@+id/iv_pic"
      android:layout_alignEndOf="@+id/iv_pic" />
  </RelativeLayout>

AFTER (✅ Correct):
  <FrameLayout>
    <ImageView android:id="@+id/iv_pic" ... />
    <View
      android:layout_gravity="bottom|end"
      android:layout_marginBottom="-5dp"
      android:layout_marginEnd="-5dp" />
  </FrameLayout>

BENEFIT: Simpler, cleaner, no invalid attributes!


EXAMPLE 2: Profile Card (ConstraintLayout)
────────────────────────────────────────────

BEFORE (❌ Invalid + Complex):
  <RelativeLayout>
    <ImageView android:id="@+id/iv" ... />
    <View android:layout_alignBottomOf="@+id/iv" ... />
    <LinearLayout android:layout_toEndOf="@+id/iv" ... />
  </RelativeLayout>

AFTER (✅ Correct + Flexible):
  <ConstraintLayout>
    <ImageView
      android:id="@+id/iv"
      app:layout_constraintStart_toStartOf="parent"
      app:layout_constraintTop_toTopOf="parent" />
    <View
      app:layout_constraintEnd_toEndOf="@+id/iv"
      app:layout_constraintBottom_toBottomOf="@+id/iv" />
    <LinearLayout
      app:layout_constraintStart_toEndOf="@+id/iv"
      app:layout_constraintEnd_toEndOf="parent" />
  </ConstraintLayout>

BENEFIT: Clearer constraints, no ambiguity!

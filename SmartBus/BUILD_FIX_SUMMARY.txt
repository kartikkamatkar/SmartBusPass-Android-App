# SmartBus Build Fix - Quick Reference

## What Was Wrong?
Android resource compiler failed because **RelativeLayout attributes were incorrect**.

## What Was Fixed?

### Three XML Files Updated:

| File | Line | Fix |
|------|------|-----|
| `fragment_profile.xml` | 85-86 | `layout_alignEnd` → `layout_alignEndOf` |
| `activity_track_bus.xml` | 115-116 | `layout_alignTop` → `layout_alignTopOf` |
| `activity_profile.xml` | 95-96 | `layout_alignBottom` → `layout_alignBottomOf` |

## Why?
When a view in RelativeLayout needs to align with **another view**, use:
- `layout_align[DIRECTION]Of="@+id/reference_view"`

When aligning with **parent**, use:
- `layout_alignParent[DIRECTION]="true"`

## Example:
```xml
<!-- ✅ CORRECT - aligning to another view -->
<View
    android:layout_alignTopOf="@+id/other_view"
    android:layout_alignEndOf="@+id/other_view" />

<!-- ❌ INCORRECT -->
<View
    android:layout_alignTop="@+id/other_view"
    android:layout_alignEnd="@+id/other_view" />
```
